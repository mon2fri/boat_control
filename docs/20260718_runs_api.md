# Comparison and Validation Execution API

## Endpoint

### `POST /api/runs/execute/`

Executes comparison and validation on two CSV files.

**Request:**
```json
{
  "file_a_path": "/path/to/uploads/abc123_file_a.csv",
  "file_b_path": "/path/to/uploads/def456_file_b.csv",
  "target_columns": ["score", "status"],
  "filters": [
    {"column": "status", "operator": "eq", "filter_value": "active"}
  ],
  "rule_ids": ["R001", "R002"]
}
```

**Response (200):**
```json
{
  "comparison": {
    "total_rows_a": 120000,
    "total_rows_b": 120000,
    "rows_with_changes": 1500,
    "total_attribute_changes": 3200,
    "row_details": [
      {
        "row_index": 42,
        "key_columns": {"id": "12345"},
        "attribute_changes": [
          {"column": "score", "file_a_value": "10", "file_b_value": "15"}
        ],
        "change_count": 1
      }
    ]
  },
  "validation": {
    "total_violations": 50,
    "violations_by_rule": {
      "R001": [
        {
          "row_index": 10,
          "rule_id": "R001",
          "rule_name": "Active Check",
          "key_columns": {"id": "100"},
          "details": "Violated R001: Active Check"
        }
      ]
    },
    "violation_count_by_rule": {"R001": 50}
  },
  "common_columns": ["id", "name", "status", "score"],
  "target_columns": ["score", "status"],
  "filters_applied": [
    {"column": "status", "operator": "eq", "filter_value": "active"}
  ]
}
```

## Processing Logic

### Comparison

1. Joins both DataFrames on the first common column (key)
2. Compares target columns row-by-row
3. Reports attribute changes with original values
4. Uses full outer join to catch rows unique to either file

### Validation

1. Loads all rules (or specified rule IDs)
2. Checks each row against rule conditions and logic
3. Reports violations with row index and key columns
4. Supports operators: eq, neq, contains, ncontains, gt, lt, gte, lte

### Filter Application

Filters are applied before comparison/validation:
- `eq`: Exact match
- `neq`: Not equal
- `contains`: String contains
- `ncontains`: String does not contain

## Performance

- Uses Polars lazy evaluation for initial filtering
- Materializes DataFrames only for comparison/validation
- Suitable for 120k x 200 datasets
- Deterministic results for same inputs
